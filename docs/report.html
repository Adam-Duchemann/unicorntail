The idea

Apply ponytail's first question to ponytail

Ponytail (74.6k★) taught AI agents to stop over-building: before writing code, walk a ladder — does this need to exist, is it already in the codebase, does the stdlib or the platform cover it — and stop at the first rung that holds. It works. It is also ~6KB of rules re-injected by lifecycle hooks into every session and every subagent, plus a second instruction system the agent must referee against the one already in CLAUDE.md.

Unicorntail is what's left after asking ponytail's own first question — does this need to exist? — of each piece of it. The ladder survives, sharpened. The hooks, the mode-state files, the per-agent re-injection, and the instruction conflicts don't: the rules are rewritten as one pre-reconciled section of the global CLAUDE.md, which already reaches every session and subagent and survives compaction natively. Still a pony's tail — just on a mythical creature.

The rules

Seven rungs, stop at the first that holds

  1. 1Doesn't need to exist? Speculative need = skip it, say so in one line.
  2. 2Already in the codebase? Reuse the helper — re-implementing what lives a few files over is the most common slop.
  3. 3Stdlib does it? Intl over a date lib.
  4. 4Native platform covers it? The platform’s built-in over a new lib (<dialog> over a modal component), CSS over JS, a DB constraint or RLS policy over app-side checks.
  5. 5Installed dependency solves it? Never add a new one for what a few lines can do.
  6. 6One line? One line.
  7. 7Only then: the minimum code that works.

Plus hard rules the evals enforce: exactly one implementation, no single-use factories, at-most-once operations are atomic, trust boundaries stay guarded however "minimal" the ask, every shortcut marked with its ceiling (// ponytail: <what>, ceiling: <limit>, upgrade when <trigger>) — and process is never on the ladder: test-coverage, i18n, and verification policies always apply in full.

Ponytail re-injects its ~1,450-token rule body at session start and into each subagent. Unicorntail rides the CLAUDE.md channel that already exists — on a 31-agent review workflow that's ~45,000 tokens that are simply never spent.

2 rule systems → 1

No mid-task refereeing

Ponytail's "YAGNI applies to tests too" and three-line output rule can conflict with the test-coverage gates and evidence-backed reporting you already enforce. Unicorntail resolves the conflicts once, at authoring time — the agent never burns reasoning on arbitration.

Nothing to break

No lifecycle hooks, no Node dependency, no mode-state files, and the rules can't silently drop off after compaction. The failure surface is a block of text.

The safety gap is the headline. On the money-moving endpoint, both the bare model and ponytail wrote a coupon redeem that double-credits under concurrency in every single run — ponytail's guardrails are abstract prose ("error handling that prevents data loss"). Unicorntail's are concrete patterns (mark-consumed atomically, conditional UPDATE … WHERE used_at IS NULL) and went 3/3. Credit where due: ponytail aces the shortcut-marking case — the ponytail: comment convention is its invention, and unicorntail keeps it.

Fairness note. Unicorntail was iterated four times against these rubrics; ponytail ran cold, no iterations. Same injection mechanism, same judge, same rubrics for all arms — but treat the 19 vs 22 gap as "tuned vs untuned on this suite", not a universal ranking. The 0/3 safety result, though, matches ponytail's design: its rules simply don't encode atomicity.

The one recurring miss is Haiku-specific. Roughly one debounce run in three still reaches for a debounce(fn, delay) factory despite an explicit ban, observed 1/3–3/3 across rolls on Haiku — the numbers above are reported as rolled, not re-rolled until green. On Sonnet — the cheapest tier that should be writing real code anyway — the shipped text passes clean: 24/24.

Tier check. Sonnet subject on the hardest case (output-shape): unicorntail 3/3, no tool 2/3. Haiku shows residual variance beyond just debounce — the single-use-factory habit surfaced 3/5 in one v4 roll — acceptable because in the house model-routing policy Haiku never writes shipped code; Sonnet is the implementation floor, and it's clean there at 24/24.

How it was built

Test-driven, four iterations to green

The eval suite came first — 8 cases with fixed rubrics, run RED against the bare baseline — then the rule text was iterated until the with-arm went green. Each version taught something about how small models actually read rules:

  1. v112/24Polite prose. The rules verifiably reached the model — and changed almost nothing. Guidance isn't behavior.
  2. v221/24Hard rules: "deliver exactly ONE implementation", explicit factory ban. Debounce and modal fixed.
  3. v322/24At-most-once atomicity pattern (conditional UPDATE … WHERE used_at IS NULL). The money-path case went 0/3 → 3/3.
  4. v424/24ponytail: marker given as a copyable template plus a one-line pre-send self-check. Marker adoption: 0/5 → 5/5. Small models copy templates; they ignore abstract conventions.

Two rubrics were corrected along the way — once too lenient (single-use options factories slipped through), once too literal (demanding exact ceiling phrasing). Fixing tests in both directions is part of the discipline. The entire loop ran on Haiku (subject + judge) with one Sonnet spot-check; zero eval tokens at frontier pricing. The ponytail head-to-head arm ran after green, with the shipped section temporarily removed for a clean environment.

Verification

Live canaries, post-ship

  • ✓Fresh session, zero injection: the debounce prompt produced the 11-line inline solution — no factory, no alternatives. The CLAUDE.md carriage works (pre-ship baseline passed this 1 in 3).
  • ✓Subagent carriage: a probe agent in a fresh session quoted the ladder's first sentence verbatim from its own context — the rules reach spawned agents with no SubagentStart hook.
  • ✓Fixture hygiene: the unicorntail skills-dir plugin stays disabled (eval fixture only); enabling it would double-inject alongside CLAUDE.md.

Known caveats. Mid-session CLAUDE.md edits don't reach already-running sessions (snapshot at start) — restart to pick up. And since the ladder ships in CLAUDE.md, clean baseline or head-to-head arms require temporarily removing the section (the runner's documented re-baseline procedure — used for the ponytail arm above, then restored byte-identical).
